### Refinement / 再洗練
ある場所で、樽に入れ、レアリティに応じた価格を支払うと、追加効果を付与してくれる。<br>
レアリティによって付く効果の種類や値がかわる。

効果名 : #7fd6e2

rarityを取得 -> 乱数を生成 -> refinementに代入 -> いろいろ付与

=  コスト  =<br>
Normal    : 32L<br>
Rare      : 4LB<br>
Epic      : 16LB<br>
Legendary : 8EL<br>
Mythic    : 32EL<br>

### Restoration / 鑑定
宝箱や敵が落とすアイテムを鑑定する。
乱数によって*再現性*が変わる。詳しくは以下。
周回必須。頑張れ。

-- 仕組み --
基礎値が設定されており、それに乱数によって決定される倍率をかける。<br>
0%~100% になっており、\*0.30~ \*1.30 が掛け算される。<br>

< 各ステータス ><br>
 0~ 29% : #be2152<br>
30~ 49% : #f5eb4a<br>
50~ 69% : #40CF84<br>
70~ 89% : #4182fa<br>
90~100% : #b4f1f9<br>

< 合計 ><br>
 0~ 19% : #be2152<br>
20~ 34% : #f5eb4a<br>
35~ 65% : #40CF84<br>
66~ 79% : #4182fa<br>
80~100% : #b4f1f9<br>

> 参考 : Wynn Craft


そして、アイテムごとにファイルを作ってloreをmacroで付与する。

## repair_cost 
* repair_cost でアイテムの種類を判別する。特にショップ。<br>
10000~ をシステム用のアイテムにする。

まずは
```
                    "minecraft:lock": {
                        "components": {
                            "minecraft:repair_cost": 255
                        }
                    }
```
でショップ用アイテムの判別をした後、`@e[type=armor_stand]` などで分割探索をし、その後個々のやつを実行する。

glass_panel > 10000<br>
appraise > 20000<br>
refinement > 30000<br>
enchantment > 40000<br>

なお、通常取得不可のもの（不具合で残ってしまうなど）は components 内に `lock:{repair_cost: 255}` を付ける。
```
            "minecraft:lock": {
                "components": {
                    "minecraft:repair_cost": 255
                }
            }
```
